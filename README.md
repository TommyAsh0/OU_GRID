## 阶段0：战略定位与硬约束

### 0.1 策略本质

Code

```
一句话定义：

在统计上表现出均值回复特性的标的上，
利用OU过程量化回复强度，
通过ATR自适应网格捕捉价格围绕均值的振荡利润。

核心假设：
价格偏离均值后有回归的统计倾向。
OU过程是对这种倾向的一阶近似。
这个近似在大部分时间"够用"，但会周期性失效。

策略类型：中低频、均值回复、逆势
交易频率：日线级别
持仓周期：单笔3-20个交易日
预期年化：10%-25%（扣除全部成本后）
预期最大回撤：15%-30%
```

### 0.2 资金硬约束

Code

```
总资金：M

网格策略总占比：≤ 50% × M
单标的最大占比：≤ 20% × M
永久现金储备：≥ 30% × M

示例（总资金100万）：
  网格策略可用：50万
  单标的上限：20万
  永久现金：30万
  剩余20万：备用/其他策略
```

### 0.3 绝对止损线——不可协商

Code

```
第一层：单标的止损
  单标的浮亏达到该标的投入资金的 -15% → 清仓
  清仓后该标的冷却30个交易日

第二层：策略止损
  策略整体回撤达 -15% → 全部仓位减半
  策略整体回撤达 -20% → 全部清仓，暂停30个交易日

第三层：年度止损
  年度累计亏损达 -25% → 当年停止该策略

以上三层止损是无条件执行的。
不存在"我觉得会反弹所以不止损"。
```

### 2.2 数据清洗规则

Code

```
1. 停牌日：前向填充收盘价，成交量/成交额填0
2. 复权：统一使用前复权价格
3. 异常值：单日涨跌幅超过±11%（非涨跌停）标记为异常
4. 缺失值：连续停牌超过20日的区间整段剔除
5. ST期间：整段数据剔除，不参与任何计算
6. 次新股：上市不足250个交易日的不纳入
```

### 2.3 指标计算

Python

```
# 伪代码：核心指标计算

def compute_indicators(df, ma_len=60, atr_len=20):
    """
    输入：df包含 open, high, low, close 列
    输出：添加 MA, ATR, Z 列
    """
    # 1. 移动平均
    df['MA'] = df['close'].rolling(ma_len).mean()

    # 2. ATR
    df['TR'] = max(
        df['high'] - df['low'],
        abs(df['high'] - df['close'].shift(1)),
        abs(df['low'] - df['close'].shift(1))
    )
    df['ATR'] = df['TR'].rolling(atr_len).mean()

    # 3. 标准化偏离度
    df['Z'] = (df['close'] - df['MA']) / df['ATR']

    # 4. 剔除ATR=0或极小值的行（避免除零）
    df = df[df['ATR'] > df['ATR'].quantile(0.01)]

    return df
```

### 3.2 统计检验细节

Code

```
检验1：ADF检验（Augmented Dickey-Fuller）
  对象：Z_t 序列
  原假设：存在单位根（非平稳）
  要求：p-value < 0.05（拒绝原假设）
  滞后阶数：自动选择（AIC准则）
  工具：statsmodels.tsa.stattools.adfuller

检验2：KPSS检验
  对象：Z_t 序列
  原假设：序列平稳
  要求：p-value > 0.10（不拒绝原假设）
  工具：statsmodels.tsa.stattools.kpss

  为什么需要同时做ADF和KPSS：
    ADF通过 + KPSS通过 → 大概率平稳 ✅
    ADF通过 + KPSS不通过 → 结果矛盾，不选 ❌
    ADF不通过 + KPSS通过 → 结果矛盾，不选 ❌
    ADF不通过 + KPSS不通过 → 大概率非平稳 ❌

检验3：Hurst指数
  对象：Z_t 序列
  方法：R/S分析法
  要求：H < 0.45
  解释：
    H < 0.5 → 均值回复倾向
    H = 0.5 → 随机游走
    H > 0.5 → 趋势倾向
```

### 3.3 稳定性检验

Code

```
目的：确保OU参数不是"碰巧"在全样本上显著

方法：
  1. 在Z_t序列上使用滚动窗口（250日），步长20日
  2. 每个窗口独立估计 κ
  3. 得到一组 κ 估计值序列：{κ_1, κ_2, ..., κ_N}
  4. 计算变异系数：CV(κ) = std(κ) / mean(κ)
  5. 要求：CV(κ) < 0.5

解释：
  CV < 0.3 → κ非常稳定 → 优质标的
  0.3 ≤ CV < 0.5 → κ基本稳定 → 可用标的
  CV ≥ 0.5 → κ不稳定 → 放弃

额外要求：
  最近2个窗口的κ都必须为正
  （如果最近的κ为负，说明当前不处于均值回复状态）
```

## 阶段4：OU参数估计与校验

### 4.1 离散OU回归

Code

```
模型：
  Z_(t+1) = a + b × Z_t + ε_t

OLS估计得到：
  b（回归系数）
  a（截距）
  σ_ε（残差标准差）

参数转换（日频，Δt = 1）：
  κ = -ln(b)
  θ = a / (1 - b)      （长期均值，理论上应接近0）
  σ_ou = σ_ε × sqrt(-2×ln(b) / (1 - b²))

半衰期：
  HL = ln(2) / κ

稳态标准差：
  s = σ_ou / sqrt(2κ)
```

### 4.2 估计的陷阱与应对

Code

```
陷阱1：b > 1 或 b < 0
  含义：OU模型不适用
  应对：该窗口标记为无效，该标的降级

陷阱2：θ远离0
  含义：Z_t的构造可能有问题，或MA长度不合适
  应对：检查MA参数，或该标的不适合本策略

陷阱3：σ_ε随时间变化明显
  含义：波动率非平稳（GARCH效应）
  应对：可接受一定程度的变化，但如果σ_ε的CV > 0.5
       说明OU模型的常数波动率假设太强

陷阱4：残差ε_t不服从正态分布
  含义：OU模型的高斯假设不成立
  应对：检查残差的JB检验，但不作为硬性筛选条件
       （实盘中我们用ATR网格来部分吸收非正态性）
```

### 4.3 参数校验清单

Code

```
□ b ∈ (0, 1)                    → OU过程有效
□ κ > 0                         → 存在均值回复
□ |θ| < 0.5                     → 长期均值接近0
□ 5 ≤ HL ≤ 20                   → 半衰期在合理范围
□ CV(κ) < 0.5                   → κ稳定
□ 最近2个窗口κ > 0              → 当前处于均值回复状态
□ 残差自相关检验通过（Ljung-Box） → 模型充分
```

## 阶段5：网格引擎设计

### 5.1 网格结构

Code

```
中轴：MA_60（60日移动平均线）

网格层定义（以做多网格为例，即价格下跌时逐层买入）：

  层1（最浅）：买入价 = MA - 1×K×ATR
  层2：          买入价 = MA - 2×K×ATR
  层3：          买入价 = MA - 3×K×ATR
  层4：          买入价 = MA - 4×K×ATR
  层5（最深）：买入价 = MA - 5×K×ATR

对应卖出价（每层买入后的获利了结）：

  层i的卖出价 = 层i的买入价 + K×ATR
  即：回到上一层的价格时卖出

对称的做空网格（价格上涨时逐层卖出）：
  与做多网格镜像对称
  注意：A股不能裸卖空，做空网格仅适用于：
    1. 已有持仓时的减仓
    2. 融券标的（如果开通融券）
    3. ETF策略

初期建议：只做单边做多网格，不做空
```

### 5.2 网格深度校验

Code

```
计算稳态标准差：
  s = σ_ou / sqrt(2κ)

网格总深度（Z空间）：
  D = n × K

  其中：
    n = 5（层数）
    K = ATR倍数

要求：
  D ≤ 2.5 × s

如果不满足：
  选项1：减小K
  选项2：减少n到4或3
  选项3：放弃该标的（波动率相对于回复力度太弱）

示例：
  假设 s = 2.8
  K = 1.0, n = 5 → D = 5
  D/s = 5/2.8 = 1.79 ✅（< 2.5）

  假设 s = 1.5
  K = 1.0, n = 5 → D = 5
  D/s = 5/1.5 = 3.33 ❌（> 2.5）
  → 改用 K = 0.7 → D = 3.5 → D/s = 2.33 ✅
```

### 5.3 仓位方案

Code

```
方案A：等权（推荐初期使用）

  每层仓位 = 单标的总资金 / n
  
  示例（单标的20万，n=5）：
    每层 = 4万

方案B：线性递增（验证A稳定后切换）

  权重比 = 1 : 1.5 : 2 : 2.5 : 3
  归一化 = 0.1 : 0.15 : 0.2 : 0.25 : 0.3
  
  示例（单标的20万，n=5）：
    层1 = 2万
    层2 = 3万
    层3 = 4万
    层4 = 5万
    层5 = 6万

实盘第1-3个月：使用方案A
第4个月起（如果方案A表现稳定）：切换到方案B
```

### 5.4 交易信号逻辑

Python

```
# 伪代码：每日信号生成

def generate_signals(current_price, ma, atr, K, n, positions):
    """
    current_price: 当日收盘价（用于盘后计算次日挂单价）
    ma: 当日MA_60
    atr: 当日ATR_20
    K: ATR倍数
    n: 网格层数
    positions: 当前各层持仓状态 {层号: "empty"/"holding"}
    
    返回：次日限价单列表
    """
    orders = []
    grid_spacing = K * atr
    
    for i in range(1, n + 1):
        buy_price = ma - i * grid_spacing
        sell_price = buy_price + grid_spacing  # 回到上一层时卖出
        
        if positions[i] == "empty":
            # 该层未持仓 → 挂买入限价单
            orders.append({
                "layer": i,
                "side": "BUY",
                "price": round(buy_price, 2),
                "quantity": compute_quantity(i),
                "type": "LIMIT",
                "valid": "DAY"  # 当日有效
            })
        
        elif positions[i] == "holding":
            # 该层已持仓 → 挂卖出限价单
            orders.append({
                "layer": i,
                "side": "SELL",
                "price": round(sell_price, 2),
                "quantity": positions[i].quantity,
                "type": "LIMIT",
                "valid": "DAY"
            })
    
    return orders
```

### 5.5 交易成本明细

Code

```
A股交易成本（2025年标准，实盘前需确认最新费率）：

买入时：
  佣金：0.025%（万2.5，大部分券商可谈到）
  过户费：0.001%（沪市）
  滑点估计：0.05%
  买入总成本 ≈ 0.076%

卖出时：
  佣金：0.025%
  过户费：0.001%
  印花税：0.05%（2024年减半后的税率，需确认）
  滑点估计：0.05%
  卖出总成本 ≈ 0.126%

双边总成本 ≈ 0.2%

最小有效网格宽度：
  每层利润 = K × ATR（Z空间中的1个网格宽度）
  换算为价格空间：Δ_price = K × ATR
  要求：Δ_price / price > 0.4%（双边成本的2倍）
  
  如果 ATR/price = 2%（典型值），K = 1.0：
    Δ_price / price = 2% → 远大于0.4% ✅
  
  如果 ATR/price = 0.3%（极低波动），K = 0.5：
    Δ_price / price = 0.15% → 小于0.4% ❌
    → 该标的不适合网格策略
```

## 阶段6：Regime检测与策略开关

### 6.1 为什么必须有这个模块

Code

```
网格交易的本质是逆势操作：
  价格跌 → 买入
  价格涨 → 卖出

当市场处于强趋势时：
  持续下跌 → 不断买入 → 越买越套 → 爆仓
  
OU模型假设κ>0（存在回复力），但：
  市场会周期性进入κ≈0甚至κ<0的状态
  
Regime检测的目的：
  在κ失效之前关闭策略，而不是等到爆仓才反应
```

### 6.2 三重检测机制

Code

```
检测器1：滚动Hurst指数
  窗口：60日
  计算频率：每日更新
  判断：
    H < 0.45 → 均值回复状态 → 正常运行 🟢
    0.45 ≤ H ≤ 0.55 → 不确定状态 → 仓位减半 🟡
    H > 0.55 → 趋势状态 → 暂停网格 🔴

检测器2：滚动ADF检验
  窗口：120日
  计算频率：每周一更新（计算量较大）
  判断：
    p-value < 0.05 → 均值回复显著 → 正常运行 🟢
    0.05 ≤ p-value ≤ 0.10 → 边缘 → 仓位减半 🟡
    p-value > 0.10 → 均值回复消失 → 暂停网格 🔴

检测器3：波动率爆发检测
  指标：ATR_5 / ATR_60（短期ATR / 长期ATR）
  计算频率：每日更新
  判断：
    比值 < 1.5 → 正常波动 → 正常运行 🟢
    1.5 ≤ 比值 < 2.0 → 波动放大 → 仓位减半 🟡
    比值 ≥ 2.0 → 波动率爆发 → 暂停网格 🔴
```

### 6.3 综合决策规则

Code

```
每日对三个检测器取最严格的信号：

  全部🟢 → 正常运行（满仓网格）
  任一🟡 → 仓位减半（新信号暂停，已有持仓保留）
  任一🔴 → 暂停网格（不再开新仓，已有持仓设置紧急止损）

暂停后恢复条件：
  连续5个交易日全部🟢 → 恢复运行
  （防止频繁切换）
```

## 阶段7：风控系统

### 7.1 风控层级

Code

```
层级1：网格级别
  单层最大亏损 = 层仓位 × 15%
  触发后：该层强制平仓

层级2：标的级别
  单标的总浮亏 > 标的投入资金的15%
  触发后：该标的全部平仓，冷却30日

层级3：策略级别
  策略总回撤 > 15% → 全部仓位减半
  策略总回撤 > 20% → 全部清仓，暂停30日

层级4：账户级别
  年度亏损 > 25% → 当年停止策略

执行优先级：层级4 > 层级3 > 层级2 > 层级1
```

### 7.2 特殊场景处理

Code

```
场景1：涨跌停
  跌停：卖出限价单无法成交
    → 次日开盘前重新挂单
    → 如果连续2日跌停无法卖出 → 第3日改为市价单（集合竞价）
  
  涨停：买入限价单无法成交
    → 取消订单，视为该信号作废
    → 不追涨

场景2：停牌
  短期停牌（≤5日）：冻结该标的网格，不影响其他标的
  长期停牌（>5日）：将该标的视为亏损，从策略P&L中扣除

场景3：标的被ST
  立即提交市价卖出单
  该标的永久移出标的池

场景4：除权除息
  除权日重新计算MA和ATR
  网格价格在除权日自动调整
  已有持仓数量按比例调整

场景5：成分股调整
  每季度检查标的是否仍在沪深300/中证500中
  如果被剔除：完成当前网格交易后不再开新仓
```

### 7.3 最大持仓限制

Code

```
单标的最大持仓层数 = n（全部层触发）
此时该标的仓位 = sum(w_i) 
不再接受该标的任何买入信号

同时运行的标的数上限：5只
总策略仓位上限：50% × 总资金

如果因为多只标的同时深层触发导致仓位超限：
  按Half-Life排序，优先保留HL最小的标的
  （HL小→回复更快→预期持仓时间更短）
```

## 阶段8：历史回测

### 8.1 数据划分

Code

```
总数据：2019-01-01 至 2025-12-31（7年）

划分方式：滚动前进（Walk-Forward）

  第1轮：
    训练：2019-01 至 2021-12（3年）→ 筛选标的 + 估计参数
    验证：2022-01 至 2022-12（1年）→ 选择K
    测试：2023-01 至 2023-12（1年）→ 评估

  第2轮：
    训练：2020-01 至 2022-12
    验证：2023-01 至 2023-12
    测试：2024-01 至 2024-12

  第3轮：
    训练：2021-01 至 2023-12
    验证：2024-01 至 2024-12
    测试：2025-01 至 2025-12

最终评估 = 三轮测试集结果的汇总
```

### 8.2 K值选择流程

Code

```
在验证集上：

  对 K ∈ {0.5, 0.8, 1.0, 1.2, 1.5} 分别回测
  
  计算每个K的：
    Calmar比率 = 年化收益 / 最大回撤
    Sharpe比率
    胜率
  
  选择标准：
    主指标：Calmar比率最高
    约束：Sharpe > 0.5 且 胜率 > 50%
  
  如果多个K的Calmar接近（差异<10%）：
    选择K较大的（更宽的网格 → 更鲁棒）

注意：
  K的选择在验证集上进行
  测试集只用来做最终评估，不能反向修改K
```

### 8.3 回测引擎关键要求

Code

```
必须实现：
  ✅ 使用次日开盘价模拟成交（不用当日收盘价）
     原因：信号在盘后产生，最早在次日才能执行
  ✅ 限价单成交判断：次日最低价 ≤ 买入限价 → 成交
                     次日最高价 ≥ 卖出限价 → 成交
  ✅ 涨跌停不成交：如果次日涨停/跌停且限价在涨跌停之外 → 不成交
  ✅ 停牌不成交
  ✅ 交易成本按5.5节计算
  ✅ 每日记录：持仓、现金、总资产、各层状态
  ✅ Regime检测在回测中也必须运行（回测也要有策略开关）

绝对不能做的：
  ❌ 使用未来数据（MA和ATR只能用当前及之前的数据）
  ❌ 当日信号当日成交
  ❌ 忽略停牌和涨跌停
  ❌ 回测结果不好就调参数重新跑（只能在验证集上调）
```

### 8.4 绩效评估

Code

```
核心指标：

  年化收益率 = (终值/初值)^(252/交易日数) - 1
  最大回撤(MDD) = max(peak_to_trough / peak)
  Calmar比率 = 年化收益 / MDD
  Sharpe比率 = (年化收益 - 2.5%) / 年化波动率
  胜率 = 盈利交易次数 / 总交易次数
  盈亏比 = 平均盈利金额 / 平均亏损金额

风险指标：

  最长回撤持续天数
  95% CVaR
  单标的最大亏损
  止损触发次数
  Regime暂停总天数

通过标准（测试集上）：

  Calmar > 0.8         → 收益相对于回撤够高
  Sharpe > 0.8         → 风险调整后收益合格
  胜率 > 55%           → 大部分交易盈利
  盈亏比 > 1.2         → 盈利交易的平均利润大于亏损
  最长回撤 < 60交易日  → 不会长期深套
  止损触发次数 ≤ 2次/年 → 极端情况不频繁

如果测试集不通过：
  不要调参数 → 回到阶段3重新审视标的筛选逻辑
  或者接受"这个策略在当前市场环境下不适用"的结论
```

### 8.5 结果可视化（bt_view.py）

把「单只标的」的一次回测画成一张**可交互的 HTML 图表**，直观核对策略行为。
绘图逻辑集中在 `src/backtest/visualizer.py`，入口脚本为根目录的 `bt_view.py`。

图表布局：

```
主图（上，占 70%）：K 线（蜡烛图）+ MA 网格中轴
                    + 每个买入点（▲ 红）/ 卖出点（▼ 绿）
                    悬浮可见：层号 / 成交价 / 数量 / 实现盈亏
副图（下，占 30%）：每天「盘前可见」的因子值
                    Z 偏离度主曲线 + Z=0 参考线
                    Regime 状态背景色带（黄=减半 / 红=暂停）
                    悬浮可见：滚动 κ / 半衰期 / Regime 名称
```

核心原则——**盘前可见、无前视**：

```
真实交易中，t 日开盘前只能看到「t-1 日收盘」为止的数据。
因此副图每日因子（Z / 滚动κ / 半衰期 / Regime）都按「每天滚动重算」：
  - 逐日推进，每天只喂入「当日及之前」的数据估计因子（不追求速度，逻辑直白）；
  - 再整体后移一位（shift 1）对齐到「盘前可观测日」。
最终：图上 t 日副图显示的，就是 t 日盘前你真实能看到的值（t 日看到的是 t-1 数据）。
买卖点与回测引擎完全同源（直接取 BacktestEngine 的成交列表 fills），保证图实一致。
```

运行方式：

## 附录A：数学推导速查

### OU过程

Code

```
连续形式：
  dZ_t = -κZ_t dt + σ dW_t

离散形式（Δt=1）：
  Z_(t+1) = e^(-κ) Z_t + σ√((1-e^(-2κ))/(2κ)) ε_t
  
  近似（κ较小时）：
  Z_(t+1) ≈ (1-κ)Z_t + σε_t

稳态分布：
  Z ~ N(0, σ²/(2κ))

稳态标准差：
  s = σ / √(2κ)

自相关函数：
  ρ(τ) = e^(-κτ)

半衰期：
  HL = ln(2) / κ
```

### 参数估计

Code

```
OLS回归 Z_(t+1) = a + b Z_t + ε_t

  κ = -ln(b)
  θ = a / (1-b)
  σ_ou = std(ε) × √(-2ln(b) / (1-b²))
```
